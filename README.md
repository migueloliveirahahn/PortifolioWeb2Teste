# 🎯 1. Objetivo do Trabalho

Desenvolver e publicar uma **página web completa, organizada e visualmente bem construída**, utilizando:

- HTML5
- CSS3
- VS Code
- GitHub
- GitHub Pages

Este trabalho tem como foco consolidar:

- estrutura de páginas web;
- organização de projetos;
- uso correto de HTML;
- estilização com CSS;
- publicação real na internet.

---

# ⚠️ 2. Regra Importante

✔ O projeto deve ser **100% estático (HTML + CSS)**

---

# 🧩 3. Tema do Projeto

Criar uma página chamada:

> **Meu Portfólio Web**
> 

A página deve representar o aluno profissionalmente, como se fosse um site real.

---

# 🏗️ 4. Estrutura Obrigatória da Página

---

## 🔹 4.1 Cabeçalho (Header)

Deve conter:

- nome do aluno (destaque);
- título profissional (ex: “Estudante de Programação Web”);
- menu de navegação com links internos.

---

## 🔹 4.2 Seção “Sobre Mim”

Deve conter:

- texto de apresentação;
- idade;
- curso;
- objetivos profissionais.

---

## 🔹 4.3 Seção “Conhecimentos”

Deve conter lista ou cards com:

- HTML
- CSS
- Git/GitHub
- VS Code
- lógica de programação

---

## 🔹 4.4 Seção “Projetos”

Mesmo que sejam simulados, deve conter:

- pelo menos 3 projetos
- título
- descrição
- status (ex: em desenvolvimento)

---

## 🔹 4.5 Seção “Contato”

Deve conter:

- e-mail
- cidade
- redes (se houver)
- formulário visual (apenas layout, sem funcionamento)

---

## 🔹 4.6 Rodapé

Deve conter:

- nome
- turma
- disciplina
- ano

---

# 🧱 5. Requisitos Técnicos

---

## ✔ 5.1 Estrutura HTML correta

Obrigatório uso de:

- `<!DOCTYPE html>`
- `<html>`
- `<head>`
- `<body>`
- `<header>`, `<section>`, `<footer>` (preferencial)
- títulos (`h1 até h3`)
- listas (`ul`, `li`)
- links (`a`)
- imagens (`img`)

---

## ✔ 5.2 Organização do Projeto

```bash
portfolio/
│── index.html
│── css/
│   └── style.css
│── img/
│   └── imagens
```

---

## ✔ 5.3 CSS Obrigatório

O aluno deve aplicar:

- cores
- tipografia
- espaçamento (margin/padding)
- alinhamento
- layout organizado
- menu estilizado
- botões
- efeitos hover

---

## ✔ 5.4 Navegação Interna

O menu deve funcionar com âncoras:

```html
<a href="#sobre">Sobre</a>
```

---

## ✔ 5.5 Responsividade Básica

A página deve:

- não “quebrar” no celular
- manter leitura organizada

---

# 🎨 6. Elementos Obrigatórios (nível mais alto)

A página deve conter:

- ✔ imagem (perfil ou ilustrativa)
- ✔ pelo menos 3 “cards” de conteúdo
- ✔ botão estilizado
- ✔ lista organizada
- ✔ uso de `<hr>` ou separadores
- ✔ efeitos `:hover`
- ✔ layout visual organizado

---

# 🧪 7. Trabalho Prático (Entrega)

---

## 📌 O aluno deverá entregar:

- link do repositório GitHub
- link do GitHub Pages
- print da página funcionando

---

# ⚙️ 8. Git – Processo Obrigatório

O aluno deve realizar:

```bash
git config --global user.name "Seu Nome"
git config --global user.email "seuemail@email.com"
git add .
git status
git commit -m "primeiro commit"
git push -u origin main
```

---

# 🌐 9. Publicação no GitHub Pages

Passos:

1. entrar no repositório
2. abrir **Settings**
3. ir em **Pages**
4. selecionar branch `main`
5. salvar
6. acessar o link gerado
